# SentinelXPrime Fix Plan

**Date:** 2026-04-04  
**Status:** Verified remediation plan for the 2026-04-04 assessment wave

## Goal

Close the verified contract, coverage, and documentation gaps without changing the suite's advisory-first posture.

## Phase 1: Metadata And Terminology Contracts

- Add `skills/using-sentinelx/agents/openai.yaml` so the bootstrap skill follows the same discovery contract as the other first-party skills.
- Expand `scripts/check-skill-metadata.mjs` so every top-level skill except `shared` requires both `SKILL.md` and `agents/openai.yaml`, plus the expected interface and policy fields.
- Clarify that `none | advisory | important | high-risk` remain the public concern levels, while `material` remains an internal notification bucket that maps to `important | high-risk`.
- Document a deterministic `risk_scope_key` shape and payload-normalization rule so active-analysis behavior is stable across implementations.

## Phase 2: Reference Depth And Stack Coverage

- Add a dedicated crypto reference covering password hashing, authenticated encryption, token signing and verification, key storage, and rotation expectations.
- Expand the stack profile set with Go, Ruby on Rails, PHP/Laravel, and Rust while keeping the same lightweight profile structure used by the existing profiles.
- Broaden the review taxonomy to include session management, CORS/CSP, XXE, error disclosure, race conditions, business logic abuse, and supply-chain exposure.
- Turn the planning checklist into concrete control questions and add explicit crypto, abuse-resistance, and compliance triggers.
- Add short, secret-free worked examples to the core reference files so the suite demonstrates expected output shape instead of only prescribing it.

## Phase 3: Validation And Regression Coverage

- Add regression coverage for missing skill metadata so contract drift is caught automatically.
- Add dedicated eval scenarios for uncertain-stage behavior and for the plan -> review -> test-rig lifecycle path.
- Add a deterministic test for `risk_scope_key` normalization and material scope rotation.
- Add static checks that confirm the new reference files, crypto cross-links, and worked examples stay in place.

## Phase 4: Documentation Hygiene

- Remove or rewrite wording that sounds like a release certification or an absolute security claim.
- Keep assessment artifacts focused on verified evidence, remaining gaps, and next actions.
- Surface the expanded stack support, crypto reference, and stage decision aid in the main README.

## Explicitly Deferred In This Wave

- dedicated GraphQL and serverless stack profiles
- large-scale performance fixtures
- Cursor and Kilo runtime validation beyond compatibility guidance
- broader finding-schema expansion beyond what is needed for terminology clarity
- community scaffolding or profile generators

## Success Criteria

- The metadata validator fails on missing or incomplete skill metadata.
- Static validation covers the new reference and documentation guardrails.
- The reference set contains both deeper coverage and worked examples.
- The eval catalog includes uncertain-stage and full lifecycle routing coverage.
- Assessment and fix-plan documents no longer read like release certification artifacts.
